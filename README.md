# SnapReport — Snaphomz Hackathon MVP

SnapReport is a minimal, hackathon-ready MVP that generates short, conversational, AI-powered monthly market reports for real estate agents.

This project demonstrates a small end-to-end stack:

- React frontend (Vite) where an agent can choose a ZIP and generate a report.
- FastAPI backend that serves market data, calls Gemini (if configured), and returns/generates a PDF using ReportLab.

This repo was built during the Snaphomz hackathon to show a fast, deployable demo.

---

## Features (MVP)

- Select a ZIP code (mock data for several ZIPs).
- Click `Generate Report` to get market metrics and conversational AI insights.
- Preview results in the browser and download a branded PDF (`SnapReport_<ZIP>.pdf`).
- Uses mock market data so you can demo without external APIs.

---

## Project Structure

- `backend/`
	- `main.py` — FastAPI app, mock market-data layer, Gemini integration, PDF generation (ReportLab).
	- `requirements.txt` — Python dependencies.
	- `.env` — local environment variables (ignored in git).
- `frontend/` — Vite + React app with a single `App.jsx` UI.

## Quick Start (Local)

Requirements: Python 3.10+, Node 18+, and git. Use the provided `venv` if present.

1) Backend

```bash
# create a virtual env (if not created already)
python3 -m venv venv
source venv/bin/activate

# install dependencies
pip install -r backend/requirements.txt

# Set GEMINI_API_KEY in backend/.env (or export in shell)
# backend/.env:
# GEMINI_API_KEY=your_real_gemini_api_key_here

# run backend
cd backend
uvicorn main:app --reload --port 8000
```

Backend will run on `http://localhost:8000`.

2) Frontend

```bash
cd frontend
npm install
npm run dev
```

Open the Vite URL (usually `http://localhost:5173`) and use the UI.

## Environment Variables

- `GEMINI_API_KEY` — (optional) If set, the backend will call Gemini (via google-generativeai client) to produce AI insights. Locally place in `backend/.env`. On Vercel, add it via the Project Settings -> Environment Variables.

Notes:
- If `GEMINI_API_KEY` is not provided, the backend will return a placeholder AI text so the UI still works for demos.

## Gemini (AI) integration

The backend calls Gemini only if `GEMINI_API_KEY` is present. Prompt engineering in `main.py` is already tuned to produce short, conversational, agent-like outputs split into four sections:

- `Market Summary:`
- `Buyer Insights:`
- `Seller Insights:`
- `Recommendations:`

If your API key encounters model availability errors, the code auto-detects supported models and you can switch `genai.GenerativeModel(...)` to a supported name (e.g., `gemini-2.5-flash`). The server prints helpful messages if models are not supported by your key.

## PDF Generation

Generated by ReportLab in `backend/main.py`. The PDF includes:

- SnapReport branding header
- ZIP code and market data
- AI insights (the four sections)

The endpoint `POST /download-report` accepts the payload returned by `/generate-report` and returns a PDF response.

## Deployment Notes (Vercel)

Backend options:
- Keep the backend local and deploy only the frontend to Vercel (frontend calls local backend during dev).
- Or deploy the backend as a separate service (e.g., a small VPS, Railway, or Cloud Run). Vercel's serverless Python options are limited — for a quick demo you can host the backend on Railway/Render or use a lightweight cloud VM.

If you deploy the backend on a host that supports env vars, add `GEMINI_API_KEY` to the host's environment (do NOT commit `.env`). On Vercel, set env vars in the Project Settings; Vercel will inject them during builds and runtime for serverless functions.

## How to let users choose from multiple ZIPs (UI / Backend hints)

Currently the frontend accepts manual ZIP input. To present a pre-defined list of ZIPs:

- Add a small `GET /zips` endpoint in `backend/main.py` that returns the available ZIPs and labels.
- Replace the input in `frontend/src/App.jsx` with a `<select>` dropdown populated from `/zips`.

Example backend snippet:

```py
AVAILABLE_ZIPS = {
	"90210": "Beverly Hills, CA",
	"10001": "Manhattan, NY",
	"94105": "San Francisco, CA",
	"33139": "Miami Beach, FL",
	"60611": "Chicago, IL",
	"02139": "Cambridge, MA",
}

@app.get('/zips')
def list_zips():
	return AVAILABLE_ZIPS
```

Then fetch that list and render a select box in the React app.

## Security & Limitations

- This MVP intentionally avoids authentication, persistence, or scheduling for speed — do not use it as-is in production.
- Validate and bound AI text length before PDF generation to avoid DoS from large payloads.

## Common Interview Questions (and suggested answers)

1. How would you swap mock data for a real real-estate data provider?

- Answer: Implement a `MarketDataService` interface and use FastAPI dependency injection to swap the mock implementation for a real HTTP client that calls Zillow/Redfin/ATTOM. Keep the `get_mock_market_data` signature the same so routes don't change.

2. How would you handle long-running AI requests at scale?

- Answer: Move heavy work to a background worker (Celery/RQ) and return a task id. Use polling / SSE / WebSockets for updates. Use caching and rate limits.

3. How do you prevent prompt injection or hallucinations?

- Answer: Sanitize user inputs, keep a controlled prompt template, and prefer structured outputs (JSON schema) when possible. Validate AI outputs with simple heuristics and fallback flows.

4. How will you manage secrets for deployment?

- Answer: Use environment variables provided by the hosting platform (Vercel, Railway, Render). Never check secrets into git. Locally, use `backend/.env` and `python-dotenv` during development.
